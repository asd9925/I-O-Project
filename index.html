Interactive Soundboard

Interface Lab I/O Project

July 2025

An interactive device that uses push buttons to trigger different segments of an EDM song




Components

  • FX Soundboard/Jack
  • Push buttons
  • Breadboard
  • Male to male wires
  • Arduino
  • Speaker
  • Force sensing resistor (discarded)
  • MicroSD Breakout Board (discarded)

Process


Attempt 1

My initial idea was for a user to press on a force sensing resistor and emit different audios through a MicroSD Breakout Board based on the force applied. I began by wiring my speaker and force resistant sensor to my breadboard. To test changing sound based on different pressures, I used the notes given in the ITP lab. I coded my Arduino to read the analog values of the force sensor and play the corresponding tone through the speaker.

Next I formatted my micro SD card to FAT 32 so it could be read by the Arduino using the software "SD Card Formatter." I then took the song "Ecstasy of Soul" by Zeds Dead, and clipped it into three segments.

I imported the three clips to my micro SD card, and inserted it into my Micro-SD Breakout Board. I then wired the breakout board pins to my Arduino pins, and wired the breakout board to ground and 3.3V. Next I used some example code given by Arduino to ensure that the Arduino was correctly reading my micro SD and its imported audio clips.



While I was able to get different tones to play through a standard speaker based on the amount of pressure applied, I realized the sensor couldn't connect to the custom audio in my MicroSD Breakout Board. It was time to pivot to an alternate audio device, so I began working with the FX Soundboard.


Attempt 2

I began the process of importing my .wav files onto the hard drive of my FX Soundboard, which unfortunately took about two hours before realizing there was a Mac update prohibiting this. I checked out a Windows computer and was successfully able to download my three .wav files to my soundboard.

I updated my code to reflect the change in method, which included getting rid of my SD card code, declaring digital pins for the soundboard, and updating my if statements to call the audio files from the FX Soundboard. I connected wires from the pins on my FX Soundboard to the pins on my Arduino, and connected the soundboard to power and ground.

I used Serial.println statements in my code for each if statement to ensure that the force sensor read to the amount of pressure the user was applying. I then connected my FX Soundboard to a JBL speaker using an AUX cord, and uploaded my code to my Arduino. Unfortunately, while my force sensor was correctly reading how much force the user applied, audio clips did not play out of the speaker unless the digital Arduino pins were moved to ground. It was time to pivot yet again.



Attempt 3

At this point, I was at a loss as to why my force sensor wouldn't connect to my FX Soundboard, so I decided to remove the force sensor logic and instead use three push buttons. This worked much better! I wired three buttons to my breadboard, each connected to a different audio clip. I updated my code to check each button pin, and if pressed, the Arduino sends a trigger to the FX Soundboard.

When the first button is clicked, the beginning of the song will play, if the second button is clicked, the build up will play, and if the third button is clicked, the beat drop will play out of the soundboard. This worked well enough considering the small amount of time I had left to complete this project.




Final Code


Below is the Arduino code I used for the final product, where each of the three buttons trigger a different audio clip from the FX Soundboard.


const int button1Pin = 12;
const int button2Pin = 11;
const int button3Pin = 10;
void setup() {
pinMode(button1Pin, INPUT);
pinMode(button2Pin, INPUT);
pinMode(button3Pin, INPUT);
pinMode(2, OUTPUT);
pinMode(3, OUTPUT);
pinMode(4, OUTPUT);
Serial.begin(9600);
digitalWrite(2, HIGH);
digitalWrite(3, HIGH);
digitalWrite(4, HIGH);
}
void loop() {
  if(digitalRead(button1Pin) == HIGH){
    Serial.println("Playing pin 2 audio");
    digitalWrite(2, LOW);
    digitalWrite(3, HIGH);
    digitalWrite(4, HIGH);
    delay(100);
  } 
  if(digitalRead(button2Pin) == HIGH){
    Serial.println("Playing pin 3 audio");
    digitalWrite(3, LOW);
    digitalWrite(2, HIGH);
    digitalWrite(4, HIGH);
    delay(100);
  }
  if(digitalRead(button3Pin) == HIGH){
    Serial.println("Playing pin 4 audio");
    digitalWrite(4, LOW);
    digitalWrite(2, HIGH);
    digitalWrite(3, HIGH);
    delay(100);
  }
  delay(50);
}


Limitations


If I knew originally that I would be using push buttons to trigger audio instead of a force sensor, I would have liked for each button to trigger a different song as opposed to a different clip from the same song. This format would make more sense for my new input method. Unfortunately I did not have enough time to update my audio clips and re upload them to my FX Soundboard.

I also wish that I could click a different button to have an audio clip stop playing a start a new one, but the FX Soundboard forces the full audio clip to play before a new one can begin.

Also if I had had more time, I would have loved to create a housing for my breadboard so that users can only see the push buttons. Additionally, given more time, I would have painted the album art onto this housing.

While I ran into many problems and bugs throughout this project, it ended up sort of working how I intended!
